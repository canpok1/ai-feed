# ドキュメント作成ルール

このルールは Markdown ファイル（README を除く）に適用されます。
詳細は [docs/07_document_rules.md](../../docs/07_document_rules.md) を参照してください。

## 基本原則

- **すべての文書は日本語で記述**
- 技術用語は英語のまま使用可（例: API, HTTP, JSON）
- 5W1H（What, Why, How, Where, When, Who）を適切にカバー

## 品質基準

| 基準 | 説明 |
|------|------|
| 明確性 | 専門知識がなくても理解できる |
| 簡潔性 | 必要最小限の言葉で表現 |
| 構造化 | 論理的な順序で整理 |
| 正確性 | 技術的な誤りがない |

## プルリクエスト説明文

```markdown
## 概要
[変更内容の簡潔な要約（1-3文）]

## 変更内容
- `ファイル名`: 変更内容の説明

## 関連Issue
fixed #(issue番号)
```

**注意**: PRタイトルにはissue番号を含めない

## GitHub Issue 説明文

```markdown
## 問題・要望
[何が問題か、何を実現したいか]

## 詳細
[背景情報や具体的な状況]

## 期待される結果
[対応後にどうなるべきか]
```

## 避けるべき表現

- 「いくつかの」「多分」 → 具体的な数値や確定的な表現
- 過度に長い文章 → 1文は2行以内を目安
- 受動態の多用 → 能動態を優先
